  <name>Task 1: Update package.json metadata and install semantic-release devDependencies</name>
  <files>package.json</files>
  <action>
Update `package.json` with the following changes (preserve all existing fields not mentioned here):

1. Update `keywords` array to exactly: `["claude-code", "anthropic", "agents", "hooks", "skills", "cli"]`
   (Current value is missing `anthropic`, `agents`, `hooks`, `skills` — per locked decision PUB-01 from CONTEXT.md)

2. Add `repository` field:
   ```json
   "repository": {
     "type": "git",
     "url": "https://github.com/anthropics-community/cc-templates.git"
   }
   ```

3. Add `bugs` field:
   ```json
   "bugs": {
     "url": "https://github.com/anthropics-community/cc-templates/issues"
   }
   ```

4. Add `homepage` field:
   ```json
   "homepage": "https://github.com/anthropics-community/cc-templates#readme"
   ```

5. Add `publishConfig` field (enables npm provenance attestation with OIDC trusted publishing):
   ```json
   "publishConfig": {
     "provenance": true
   }
   ```

6. Update the `files` array to include `LICENSE`:
   ```json
   "files": ["bin/", "dist/", "components.json", "README.md", "LICENSE"]
   ```

7. Add semantic-release devDependencies (run after editing package.json):
   ```bash
   npm install --save-dev semantic-release @semantic-release/commit-analyzer @semantic-release/release-notes-generator @semantic-release/npm@13.1.4 @semantic-release/github
   ```
   Use `@semantic-release/npm@13.1.4` explicitly — this is the minimum version supporting OIDC trusted publishing.

Do NOT add `@semantic-release/git` — per research recommendation, omit it for simplicity (version in package.json may lag published, which is acceptable).
Do NOT set registry-url in setup-node (this is for the CI workflow task, but noting here to avoid any `.npmrc` side effects during local install).
